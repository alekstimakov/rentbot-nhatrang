# Telegram Bot Rent

Телеграм-бот для аренды байков в Нячанге.

## Зачем нужен проект

Бот автоматизирует процесс аренды:

- клиенту не нужно писать менеджеру вручную и объяснять детали в чате;
- менеджер получает структурированные заявки и может обработать их в 1 клик;
- все заявки и настройки хранятся централизованно в базе.

Итог: быстрее обработка заявок, меньше ручной рутины, меньше потерь лидов.

## Что делает бот

### Для клиента

- показывает категории и доступные модели байков;
- принимает заявку на бронирование (даты, доставка/самовывоз, контакт);
- показывает статус и список заявок;
- даёт кнопку связи с менеджером при активной брони;
- даёт кнопку `SOS` при активной брони;
- даёт раздел `Полезная информация` при активной брони:
  - `Рекомендации по вождению` (текст + видео),
  - `Путеводитель` (текст + документ).

### Для администратора

- добавление/удаление моделей в каталог;
- подтверждение/отклонение заявок;
- просмотр заявок по статусам:
  - в ожидании,
  - активные,
  - отклонённые,
  - завершённые;
- отправка сообщения клиенту по заявке;
- настройка контента из админ-панели:
  - правила бронирования,
  - адрес офиса,
  - образец договора,
  - советы по вождению,
  - путеводитель;
- сервисные действия: удаление броней, полная очистка БД (с паролем).

## Основная логика статусов заявок

- `pending` — в ожидании решения менеджера;
- `active` — подтверждена, аренда активна;
- `rejected` — отклонена;
- `finished` — завершена.

## Технологии

- Python 3.11+
- aiogram 3.x
- SQLite (по умолчанию, файл `data/bikes.db`)

## Структура проекта (актуальная)

```text
project/
  main.py
  requirements.txt
  DEPLOY.md
  deploy/
    tg-bot-rent.service
  src/
    bot_app/
      runtime.py          # запуск, router, общие утилиты
      db.py               # слой работы с БД
      user_handlers.py    # пользовательские обработчики
      admin_handlers.py   # админские обработчики
      flows.py            # state-machine сценарии
      keyboards.py        # все клавиатуры
      texts.py            # текстовые шаблоны
```

## Переменные окружения

Минимально необходимые в `.env`:

- `BOT_TOKEN` — токен Telegram-бота;
- `ADMIN_IDS` — id админов через запятую (например `123,456`);
- `ADMIN_PASSWORD` — пароль для полной очистки базы.

Опционально:

- `DB_PATH` — путь к SQLite базе (по умолчанию `data/bikes.db`);
- `START_PHOTO_PATH` или `START_PHOTO_URL` — фото для стартового экрана.

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate    # Windows
pip install -r requirements.txt
python main.py
```
